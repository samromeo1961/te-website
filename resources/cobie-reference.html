COBie Reference Guide

Construction Operations Building information exchange - structured data handover for facility management

What is COBie?

COBie (Construction Operations Building information exchange) is an international standard (ISO 15686-4) for capturing and delivering asset data from design and construction to operations and maintenance. It provides a structured format for handover information, typically delivered as a spreadsheet with interconnected worksheets. COBie ensures the right information reaches facility managers at project completion.

Core Worksheets

The essential worksheets that form the backbone of every COBie deliverable. These contain the primary asset and spatial data.

Facility Required

Top-level information about the building or facility. Contains project name, address, phases, and site data.

Key Columns

Name CreatedBy ProjectName SiteName AreaUnits CurrencyUnit
Floor Required

Building levels and floors. Links to Facility and contains elevation and area information.

Key Columns

Name CreatedBy Elevation Height
Space Required

Rooms and spaces within floors. Contains area, function, and occupancy data for each space.

Key Columns

Name FloorName RoomTag UsableHeight GrossArea NetArea
Type Required

Asset types (product families). Contains manufacturer, model, warranty, and specification data.

Key Columns

Name Category Manufacturer ModelNumber WarrantyDurationParts ExpectedLife
Component Required

Individual asset instances. Links to Type and Space, contains serial numbers and installation data.

Key Columns

Name TypeName Space SerialNumber InstallationDate BarCode
Zone Reference

Groupings of spaces for systems or management purposes (HVAC zones, fire compartments, cleaning zones).

Key Columns

Name Category SpaceNames

Data Relationships

COBie worksheets are interconnected through key relationships. Understanding these links is essential for data integrity.

From Sheet To Sheet Relationship Link Column
Floor Facility Floors belong to a Facility Implicit (single facility)
Space Floor Spaces are located on Floors FloorName

Component Space Components are located in Spaces Space
Zone Space Zones contain multiple Spaces SpaceNames
System Component Systems contain Components ComponentNames

Job Type Jobs are defined for Types TypeName
Spare Type Spares are associated with Types TypeName

Data Integrity

All links must reference existing rows by their exact Name value. Orphaned references (pointing to non-existent rows) will cause validation failures. Always verify relationships before final delivery.

Validation Checklist

Common validation rules and quality checks for COBie deliverables.

  • ✕
    Required Fields Empty Name, CreatedBy, and key relationship columns must have values in every row
  • ✕
    Duplicate Names Each Name within a worksheet must be unique to enable proper linking
  • ✕
    Orphaned References TypeName, FloorName, SpaceNames must match existing rows exactly
  • !
    Missing Categories Type and Component Category should use standard classifications (OmniClass, Uniclass)
  • !
    Placeholder Values Values like "n/a", "TBD", or "unknown" should be replaced with actual data
  • ✓
    Valid Email Format Contact.Email must be valid email addresses (used as unique identifiers)
  • ✓
    Consistent Units AreaUnits and other unit definitions used consistently throughout
  • ✓
    Document Links Valid Document file paths or URLs should be accessible and functional
